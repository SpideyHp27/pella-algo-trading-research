## Critical lessons from today

1. **MT5 Optimization mode = disk bomb.** Twice today the C: drive went to <1 GB free because Strategy Tester defaulted to optimization mode and fanned across Market Watch symbols. **Always verify Optimization=Disabled before clicking Start.** Memory: `mt5_bridge_runtime_gotchas.md`.

2. **NDAQ ≠ NASDAQ-100.** NDAQ is Nasdaq Inc the company stock (~$91). NDX is the actual NASDAQ-100 index (~27,775). Wrong-symbol backtests run for ~10 seconds with 0 trades and look like data-quality issues. Memory: `darwinex_symbol_aliases.md`.

3. **Bridge `tester_configure` doesn't override Optimization mode** — only the explicitly-named fields. Whatever Optimization setting was last set in MT5 persists. Don't use the bridge for a fresh test config until v0.44 (or later) adds an `optimization` parameter.

4. **MC Bootstrap can mislead on low-N samples.** With 32 trades and one big winner, bootstrap-with-replacement shows wide variance (35% prob unprofitable in alternate histories) — but that's a statistical artifact of small N + skewed distribution, NOT a verdict on the strategy. The actual edge can still be real. Don't shelve based purely on a low-N MC.

5. **Trailing stop is essential for vol-expansion / breakout strategies.** v0.1 IDNR4 had 3% WR because positions ran into profit then reverted to original SL. Adding `UseTrailingStop` (lock SL at posMaxFavorable - 1×R) is the standard fix.

6. **MT5 caches grow ~2-7 GB per backtest run.** `Tester/agent/bases/` and `Terminal/.../bases/Darwinex-Demo/ticks/` are the big ones. Wipe agent caches every few runs; keep ticks for active symbols only.

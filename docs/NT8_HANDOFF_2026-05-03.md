## Project context (1-paragraph)

Pella is a multi-strategy algo trading project. Methodology: Pipeline v1.3 + López de Prado-style validation (G4a yearly stability, G4b param sensitivity, G4d holdout — the holdout is the decisive curve-fit detector). 5 strategies have survived on Darwinex MT5 CFD data. Now porting to NT8 for parallel futures validation. **Important:** futures markets have different microstructure (settlement at 4:45 PM ET, different tick sizes, different commission/slippage). Strategies that work on CFD overnight may need intraday redesign for Lucid (which prohibits overnight on funded accounts).

| Rule | Impact on strategy design |

## NinjaScript / NT8 port reference (MQL5 → NinjaScript mapping)

Common patterns the porter will need:

| MQL5 concept | NinjaScript equivalent |
|---|---|
| `OnTick()` / new-bar gate via `iTime(0)` | `OnBarUpdate()` with `Calculate = Calculate.OnBarClose` |
| `iHigh(_Symbol, PERIOD_H4, 1)` | `Highs[1][1]` (multi-TF series via `AddDataSeries(BarsPeriodType.Hour, 4)`) |
| `iATR(handle)` + `CopyBuffer` | `ATR(14)[0]` (NinjaScript indicator returns array) |
| `iMA(handle, ...)` with EMA | `EMA(50)[0]` |
| `trade.Buy(lots, sym, price, sl, tp)` | `EnterLong(quantity, "signalName")` then `SetStopLoss` / `SetProfitTarget` |
| `PositionGetDouble(POSITION_PROFIT)` | `Position.GetUnrealizedProfitLoss(PerformanceUnit.Currency, Close[0])` |
| `OrderCalcProfit` for risk math | manually compute via `Instrument.MasterInstrument.PointValue` and `TickSize` |
| `AccountInfoDouble(ACCOUNT_BALANCE)` | `Account.Get(AccountItem.CashValue, Currency.UsDollar)` |
| `MagicNumber` filter | NinjaScript strategies are isolated by class — no magic needed |
| Session filter (broker hours) | `Bars.IsFirstBarOfSession`, `Time[0].Hour`, or `IsExitOnSessionCloseStrategy = true` |
| `iCustom("indicator name")` | direct `MyCustomIndicator()` C# call |
| `OnTradeTransaction` | `OnExecutionUpdate(Execution e, ...)` |

For EOD-flatten (Lucid mandatory), set in `OnStateChange` → `State.SetDefaults`:
```csharp
IsExitOnSessionCloseStrategy = true;
ExitOnSessionCloseSeconds = 30;
```

## STRATEGY 1: ChBVIP_v0.4 (Channel Breakout VIP — Donchian breakout + vol-aware trail)

**Original source:** Zenom community MotherEA S1, ported to MT5 by Pella, hardened by us with v0.4 vol-aware trailing stop.

**Markets to port:** USDJPY → 6J futures (Yen) | XAUUSD → MGC (micro gold) or GC

**Timeframe:** H1 chart, native logic (not multi-TF)

**Logic (plain English):**
1. On each new H1 bar close, compute Donchian channel: `upBound = highest high of last 50 bars`, `downBound = lowest low of last 50 bars`.
2. When flat: place a pending **buy stop** at `upBound + 1 point` with stop loss at `downBound - 1 point` (channel-low SL).
3. **Long-only strategy** (per source EA v0.2 design — does NOT short).
4. Lot sizing: percent-risk per trade. `lots = (balance × RiskPercent%) ÷ (USD-loss-per-lot at the actual SL distance)`. Use `OrderCalcProfit` equivalent.
5. **Vol-aware trailing stop (the v0.4 feature):**
   - At each bar, compute `ATR(14) / close` as a percentage.
   - If `ATR_pct >= MinAtrPctForTrail` (default 0.20): activate ATR trailing stop = `close - 2 × ATR`; ratchet upward only.
   - If `ATR_pct < MinAtrPctForTrail`: trail goes IDLE — only the channel SL exits.
   - This prevents whipsaw exits in low-volatility regimes (which killed the strategy on USDJPY 2026 Q1 before we added the gate).
6. Daily target gate: if today's realized profit ≥ `DailyTakeProfitUSD` (default $1000), no new entries that day.
7. Per-bar SL update: `newSL = max(currentSL, channelLow, trailingStop)` — never loosen.

**Exact MQL5 inputs:**
```
Length             = 50      // Donchian length
AtrPeriod          = 14
AtrMult            = 2.0     // trailing distance in ATR units
UseFixedStopLoss   = false   // false = use ATR trail
UseChannelExit     = true    // channel-low SL
MinChannelWidthPct = 0.0     // entry filter — DEFAULT OFF
MinAtrPctForTrail  = 0.20    // vol-trail gate (0.20 = 20 basis points) — UNIVERSAL DEFAULT
UseDailyTarget     = true
DailyTakeProfitUSD = 1000
MinTradeValueUSD   = 40
UseRiskPercent     = true
RiskPercent        = 1.0     // 1% per trade for eval; 0.4-0.5% for live recovery
MaxLotsCap         = 10.0
```

**Lucid compatibility:** **needs EOD-flatten mod.** H1 channel breakouts often hold overnight. Add `IsExitOnSessionCloseStrategy = true` and accept that some trades will be cut short. Re-validate after the mod.

## STRATEGY 2: TuesdayTurnaroundNDX (TT — calendar-effect)

**Original source:** Discord/Zenom community port. Calendar-effect strategy (Mon→Wed reversal pattern).

**Markets to port:** NDX (CFD) → **NQ or MNQ** (Nasdaq-100 futures)

**Timeframe:** H1 chart (or any TF — strategy is calendar-time driven, NOT bar-driven; H4 produces bit-identical results)

**Logic (plain English):**
1. Strategy fires entries at fixed clock times. On Monday at 16:30 broker time (= ~9:30 AM ET on Darwinex), check the **Friday-red filter**:
   - Look up Friday's daily bar (D1, 3 calendar days back from Monday).
   - If `Friday close < Friday open` (Friday was red): proceed. Else skip this week.
2. Enter LONG market with risk-based sizing. **No stop loss is set** (the strategy uses a synthetic ~100-point SL only for sizing math).
3. Hold the position through Tuesday and into Wednesday.
4. On Wednesday at 20:00 broker time (= ~1:00 PM ET): close the position (market exit). One trade per week max.
5. Sizing modes: fixed lots (default) or percent-risk (1% recommended for compounding).

**Exact MQL5 inputs:**
```
InpEntryDay        = 1       // Monday
InpEntryHour       = 16
InpEntryMinute     = 30
InpExitDay         = 3       // Wednesday
InpExitHour        = 20
InpExitMinute      = 0
InpUseFridayFilter = true
InpUseMoneyManagement = true   // false=fixed lots; true=percent-risk
InpRiskPercent     = 1.0
InpMaxDailyLossPct = 5.0
InpMaxDailyTrades  = 10
```

**Lucid compatibility:** **CANNOT DEPLOY ON LUCID AS-IS.** Strategy holds Mon→Wed = multi-day. The entire edge is the multi-day calendar effect. Two paths:
- **Option A (compatible):** Convert to Mon-only intraday: open Mon afternoon, EOD-flatten same day. Loses ~70% of the historical edge but stays Lucid-compliant.
- **Option B (recommended for futures):** Skip Lucid for this strategy. Deploy on a 24-hr-allowed prop firm (FundedNext via MT5, FTMO, 5ers — all have CFD overnight).
- **Option C:** Use a longer timeframe / fewer trades and hope Lucid changes the rule (don't bet on it).

Suggest porter implements Option A as a "Lucid-compliant" variant and keep Option B as the "natural" version. Document both.

## STRATEGY 3: IDNR4_v0.3 (Inside Day + NR4 volatility expansion)

**Original source:** Street Smarts (Raschke / Connors, 1995), Chapter 19. Pella v0.3 added trailing stop + percent-risk sizing.

**Markets to port:** XAUUSD → **MGC or GC** (gold futures)

**Timeframe:** H4 chart with Daily indicator data

**Logic (plain English):**
1. Strategy fires once per new H4 bar. Reads YESTERDAY's daily bar (PERIOD_D1, shift 1).
2. **Inside Day check:** `yesterday_high < day-before_high AND yesterday_low > day-before_low` (yesterday's range entirely inside the prior day's range).
3. **NR4 check:** `yesterday_range = (yesterday_high - yesterday_low)` is the NARROWEST of the last 4 daily bars.
4. If both conditions met: place an OCO bracket (BuyStop + SellStop) for today:
   - BuyStop at `yesterday_high + 1 point` with SL at `yesterday_low - 1 point`
   - SellStop at `yesterday_low - 1 point` with SL at `yesterday_high + 1 point`
   - Both pending orders valid for one day (ORDER_TIME_DAY).
5. On fill: cancel the OCO partner. Optionally arm a **stop-reverse** order at the opposite extreme for the same day.
6. **Exits** (in priority order):
   - Channel-bracket SL (the original symmetric stop)
   - Fixed dollar stop (default $500)
   - Daily loss circuit (close all if today's P&L ≤ -$1000)
   - Trailing stop: activate at 1R unrealized profit (R = bracket width); trail at 1R below high-water
   - MOC exit: if held 2 days unprofitable, force-close
7. Lot sizing: percent-risk via `OrderCalcProfit(BUY, entry, SL)` — broker-agnostic.

**Exact MQL5 inputs:**
```
NRPeriod          = 4         // narrowest of last 4 days
UseStopReverse    = true
MaxHoldDays       = 2         // MOC exit after 2 days
FixedStopLossUSD  = 500.0
DailyMaxLossUSD   = 1000.0
MinTradeValueUSD  = 40.0
UseTrailingStop   = true
TrailingActivationR = 1.0     // activate at 1R profit
TrailingStepR     = 1.0
EntryOffsetPoints = 1
UseRiskPercent    = true
RiskPercent       = 1.0
MaxLotsCap        = 10.0
```

**Lucid compatibility:** **needs intraday redesign.** Strategy holds 1-2 days minimum. EOD-flatten kills the whole premise (volatility expansion AFTER the inside-day setup typically doesn't fully resolve same-day). Recommend deploy on FundedNext-class CFD firm or a 24-hr futures prop.

**MT5 backtest comparison numbers:**

| XAUUSD H4 (full window IS) | 136 | 1.06 | 1.91 | 11.6% |
| XAUUSD H4 (1yr holdout) | 33 | **1.45** | **2.29** | 4.95% |

## STRATEGY 4: PellaMarubozu_v0.1 (Marubozu pattern breakout)

**Original source:** Built clean-room from a marketplace .set spec (the .ex5 was license-locked). Methodology mirrors the pattern. Pella native EA.

**Markets to port:** XAUUSD → MGC | also try NQ/MNQ to see if it works on indices

**Timeframe:** M5 chart, NY session 17:00-19:00 broker time (= ~10:00 AM - 12:00 PM ET — first 2 hours of NY equity open)

**Logic (plain English):**
1. Strategy fires on each new M5 bar during the session window. Pre-checks:
   - `position_count_for_strategy < 1` (one position max)
   - `daily_loss_pct < MaxDailyLossPct` (5% default)
   - `today_trade_count < MaxDailyTrades` (10 default)
   - `current_spread_pts <= MaxSpreadPoints` (300 default)
   - `current_time` within session window
2. **Marubozu detection on the most recently closed bar (shift 1):**
   - `body = |close - open|`
   - `upper_wick = high - max(open, close)`
   - `lower_wick = min(open, close) - low`
   - `total_range = high - low`
   - **Reject if** `max(upper_wick, lower_wick) / total_range > WickMaxPercent` (0.15 default — bar must have small wicks).
   - Compute `avg_body` over last `BodyLookback` (50) closed bars.
   - **Reject if** `body < BodyFactor × avg_body` (1.5× default — body must be meaningfully larger than recent average).
   - If passes: direction = +1 (bullish) if close > open, -1 (bearish) otherwise.
3. **Trend filter (optional, recommended ON):** only trade in direction of `EMA(200)`. Long only if close > EMA, short only if close < EMA.
4. **Entry (mode 1 = breakout):** place stop order in signal direction:
   - Long: BuyStop at `signal_high + BufferPoints × point` with SL at `entry - ATR × ATRMultSL` and TP at `entry + (TP_distance = SL_distance × RR)` (default RR=3).
   - Short: mirror.
   - Order valid for at most `MaxBarsAfterSig` (3) new bars; cancel if not filled.
5. **Trailing stop:**
   - Activate when unrealized profit ≥ `TrailStartPts` (100) points.
   - Maintain SL at `current_price ± TrailDistPts` (25) — never loosen.
   - Step size `TrailStepPts` (20) — only update if change ≥ step.

**Exact MQL5 inputs (canonical config):**
```
InpBodyLookback      = 50
InpBodyFactor        = 1.5    // body > 1.5x avg
InpWickMaxPercent    = 0.15   // wick <= 15% of range
InpEntryMode         = 1      // breakout entry
InpBufferPoints      = 200
InpMaxBarsAfterSig   = 3
InpATRPeriod         = 14
InpATRMultSL         = 1.0
InpRR                = 3.0
InpUseTrendFilter    = true
InpEMAPeriod         = 200
InpUseTrailing       = true
InpTrailStartPts     = 100
InpTrailDistPts      = 25
InpTrailStepPts      = 20
InpUseSessionFilter  = true
InpSessionStartHour  = 17     // broker time = ~10am ET
InpSessionEndHour    = 19     // broker time = ~12pm ET
InpLotMode           = 2      // PctBalance
InpPctBalance        = 0.5    // 0.5% per trade (lower than other strategies — many trades)
InpMaxDailyLossPct   = 5.0
InpMaxDailyTrades    = 10
InpMaxSpreadPoints   = 300
```

**Lucid compatibility:** **NATURALLY COMPLIANT.** Intraday only, NY session, no overnight. The cleanest of the 5 to deploy on Lucid.

## Why the NT8 port worked cleanly

The original NT8 strategy uses `Calculate.OnBarClose` (closed-bar logic) which maps directly to MT5's new-bar detection in `OnTick`. Indicator values use `[1]` (previous bar) consistently — that's source-binding parity preserved.

Channel computation in MQL5 uses explicit `iHigh/iLow` loop instead of an indicator handle; this is intentional to avoid divergence vs NT8's `MAX(High, period)/MIN(Low, period)` which look back from the current bar.
